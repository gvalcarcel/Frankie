# v0.7.0 - Datos estructurados y comprensión pedagógica

Fecha de preparación: 2026-06-28

Estado: preparada y validada, pendiente de tag y publicación en GitHub.

## 1. Resumen ejecutivo

Frankie Core `v0.7.0` completa su alcance funcional offline. La release convierte los informes documentales en datos reutilizables, incorpora evidencias estructuradas, mejora Doctor como herramienta pedagógica y añade un itinerario de aula para FP Básica.

La versión sigue siendo de solo lectura. No se conecta al servidor físico Frankie, no consulta sus máquinas virtuales en tiempo real y no realiza reparaciones.

## 2. Novedades principales

- JSON estable para Status, Inventory, Audit y Doctor.
- Consulta de evidencias con `list`, `validate` y `show`.
- Modelo inicial de evidencias estructuradas y loader tolerante.
- Doctor pedagógico con severidad, urgencia, impacto y pasos seguros.
- Guías, glosario y ocho actividades para FP Básica.
- Estrategia OFFLINE/LIVE y diseño seguro de Live Mode.
- Errores CLI controlados y contratos documentados.

## 4. Ejemplos JSON

Status expone versión, modo, fuente y estado global:

```json
{
  "schema_version": "1.0",
  "command": "status",
  "frankie_core_version": "0.7.0",
  "mode": "offline",
  "overall_status": "WARNING"
}
```

Doctor mantiene el enfoque pedagógico en formato procesable:

```json
{
  "schema_version": "1.0",
  "command": "doctor",
  "frankie_core_version": "0.7.0",
  "mode": "offline",
  "overall_diagnosis": "ACTIONS_RECOMMENDED",
  "issues_reviewed": 1
}
```

Los ejemplos están abreviados. La salida real incluye componentes, checks, evidencias o diagnósticos según el comando.

## 5. Evidencias estructuradas

Frankie Core carga fichas JSON desde `docs/evidencias/structured/`.

El loader:

- valida campos mínimos;
- conserva fichas válidas si otra falla;
- informa de incidencias sin traceback;
- rechaza fichas que declaren contenido sensible;
- no usa red ni escribe archivos.

En la preparación de esta release se validaron seis evidencias y ninguna resultó inválida.

## 6. Doctor pedagógico

Doctor interpreta hallazgos activos de Audit Engine y explica:

- qué ocurre;
- severidad y urgencia;
- impacto y motivo de importancia;
- siguiente acción recomendada;
- pasos seguros;
- acciones que deben evitarse.

SMB no aparece como incidencia activa porque la evidencia vigente lo mantiene validado. Portainer conserva el único aviso conocido.

## 7. Documentación de aula

`docs/aula/` contiene:

- introducción a Frankie y Frankie Core;
- guía de uso de la CLI;
- lectura de Status y Audit;
- explicación de Doctor;
- introducción a JSON y evidencias;
- glosario técnico;
- ocho actividades con objetivo, material, pasos, resultado, preguntas y rúbrica.

El material está orientado a FP Básica y no necesita acceso a la infraestructura.

## 8. Diseño de Live Mode

Live Mode está **diseñado, pero no implementado**.

El diseño propone `live-status` y `live-audit` como futuros comandos separados. Define autorización expresa, mínimo privilegio, consultas permitidas, operaciones prohibidas, timeouts, errores seguros y saneamiento de evidencias.

Estos nombres no están registrados en la CLI de `v0.7.0` y no realizan ninguna conexión.

## 9. Limitaciones

- Los estados proceden de evidencias locales y pueden quedar desactualizados.
- No existe Live Mode real.
- No existe Repair Mode.
- No hay monitorización continua, dashboard, API o integración IA.
- La validación de evidencias no sustituye todavía una validación completa contra JSON Schema.
- La release preparada no está publicada hasta que exista tag y GitHub Release.

## 10. Riesgos conocidos

| Riesgo | Estado | Tratamiento |
|---|---|---|
| Portainer publica el puerto 8000 | `WARN / LOW` | Revisar en una Work Order LIVE específica |
| Evidencia offline desactualizada | Abierto | Mostrar fuente y actualizar mediante procesos controlados |
| Datos internos en capturas futuras | Futuro | Evidencia raw privada y saneamiento obligatorio |
| Confundir diseño LIVE con función activa | Controlado | Ayuda y documentación indican que no está implementado |

SMB permanece validado como `OK / PASS / INFO`.
