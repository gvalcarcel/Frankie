# Versioning

Frankie seguirá Semantic Versioning:

```text
MAJOR.MINOR.PATCH
```

## Major

Incrementar `MAJOR` cuando haya cambios incompatibles:

- reorganizacion incompatible de estructura,
- ruptura de contratos de scripts,
- cambio mayor de modelo de datos,
- cambio de estrategia de despliegue no compatible.

## Minor

Incrementar `MINOR` cuando se agreguen capacidades compatibles:

- nuevos scripts,
- nuevas auditorías,
- nuevas plantillas,
- nuevas secciones de knowledge base,
- nuevas integraciones.

## Patch

Incrementar `PATCH` cuando haya correcciones compatibles:

- errores de documentación,
- ajustes menores de scripts,
- mejoras de mensajes,
- correcciones de checklists,
- pequeñas mejoras de seguridad sin cambio de contrato.

## Versión actual

```text
v0.5.0 Foundation
```
